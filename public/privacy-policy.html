UI Remix Privacy Policy

Last updated: May 21, 2026

What UI Remix Does

UI Remix lets you create local rules that change the appearance of websites you visit. Rules are stored in Chrome storage on your device.

Data Stored Locally

UI Remix stores saved UI rules, rule enabled/disabled state, AI enabled/disabled state, and an optional AI access token in chrome.storage.local.

Data Sent To The AI Proxy

When AI parsing is enabled and you preview a command, UI Remix sends your command, the current page URL/domain, page title, and a capped list of visible page element metadata to the configured UI Remix backend proxy. Candidate metadata can include element tag names, selectors, text snippets, roles, labels, classes, IDs, and approximate bounding boxes.

Data Not Collected

  • UI Remix does not collect passwords, cookies, payment details, or browsing history.
  • UI Remix does not send full page HTML to the AI proxy.
  • UI Remix does not sell user data.

Backend Logs

The backend proxy logs operational metadata such as status code, latency, domain, command length, candidate count, intent, target count, and hashed IP or access-token identifiers. It does not log full commands or page text snippets.

AI Provider

The backend proxy sends minimized command context to OpenAI to produce structured rule suggestions. The OpenAI API key is stored only on the backend and is never included in the Chrome extension.

Your Controls

You can disable AI parsing in the extension popup. When AI is disabled, commands use local parsing only and do not call the backend proxy.
